    How the process works

    A usable backup is created, verified, then restored through protected checks.

    A
    Backup processPrepare a recovery artifact
    1. 1
      RequestChoose type, modules, and storage.
    2. 2
      Run workerCreate the encrypted artifact.
    3. 3
      VerifyCheck server checksum and integrity.
    4. 4
      ReadyOnly verified artifacts can be restored.
    B
    Restore processRecover through controlled validation
    1. 1
      RequestSelect a verified backup.
    2. 2
      ApproveA different admin checks it with MFA.
    3. 3
      Dry-runValidate in an isolated target.
    4. 4
      RestoreExecute with a fresh MFA check.
    5. 5
      ValidateConfirm integrity and system health.
    Safety rule: A backup request does not automatically overwrite source code or production data. Only artifacts reported by the backend as available and verified count as recovery coverage. Restore and rollback require maker-checker approval by a different authorized administrator, fresh MFA, an isolated dry-run, and final integrity checks. These technical recovery actions do not bypass HR or Payroll business approval workflows.
